# POG 伦理与数据处理说明（v0.2）

本文供路演、用户协议附件与内部评审使用；**不构成法律意见**，上线前应由法务审核。

---

## 1. 产品定位声明

POG 是**信息与关系结构化工具**，用于辅助整理、教学演示、合规培训、产品创意等场景。  
**不提供**医学诊断、心理治疗或司法鉴定结论；系统输出的特质与推断均为**假设**，需人工复核。

---

## 2. 合法与合意使用

### 2.1 谁的数据可以进入系统

| 情形 | 要求 |
|------|------|
| 用户建模**本人** | 明示知情同意；说明用途、保留期限、删除方式。 |
| 教练/咨询师整理**来访者**材料 | 遵守执业规范与合同约定；最小必要原则；建议独立 BAA/数据处理协议。 |
| 企业培训 | 仅使用**合成或已授权剧本**；真实员工数据需劳动/个人信息合规评估。 |
| 黑客松演示 | **虚构人物与合成对话**；禁止使用真实第三方隐私。 |

### 2.2 禁止用途（需在 ToS 与界面重申）

- 在**未获授权**情况下收集、推断、存储**他人**人格或私密社交图。  
- 辅助**欺诈、骚扰、歧视、人肉搜索、未授权渗透测试中的社工攻击**。  
- 将推断结果作为**招聘、信贷、保险核保**等高风险自动化决策的**唯一依据**（若涉及自动化决策，须符合当地法律与解释义务）。

**与「信用 / 声誉」相关的正向表述边界**：POG 可强调**可审计的材料编排与假设管理**（谁、在何批次、依据哪些片段得出何种推断），用于合规与研究中的**透明度附件**；但若产品对外涉及评分、排序或准入，须单独进行法律与公平性评估，**不得**将人格子图输出等同于信用评分或声誉结论。

### 2.3 「社会工程」相关表述

对外统一使用：**社工防御演练**、**安全意识教育**、**互动沟通仿真**等中性表述；技术文档内部可保留「社会工程」一词但须与**防御场景**绑定说明。

### 2.4 心理干预与临床边界

在心理科技、教练或教育场景中，POG 可作为**辅助整理与协同对齐工具**（例如多源材料合并、模式觉察、带证据路径的摘要），**不构成**心理治疗、心理咨询关系本身或医学诊断。涉及来访者或员工敏感信息时，须遵守执业规范、劳动合同与个人信息保护要求；输出应持续区分**事实观测**与**模型假设**，并提示专业复核。

**延伸阅读**：跨领域创新与效用（含信用、干预、社工）的答辩级展开见 [`PRESENTATION_SHOWCASE.md`](PRESENTATION_SHOWCASE.md) §3.4；**实现侧约束摘要**见 [`TECH_SPEC.md`](TECH_SPEC.md) **§7.2**。

## 3. 数据最小化与脱敏

- 默认不采集：精确地理位置、证件号、金融账户等与人格建模无直接必要字段。  
- 导入时提供**脱敏选项**（手机号、身份证、银行卡等正则掩码）。  
- 原始文件与图数据分级存储：原始件可加密落对象存储；图内 `Utterance.text` 可按策略截断或哈希化（实现期可配置）。

---

## 4. 保留、删除与导出

- **保留期限**：默认建议用户可配置（如 90 天未活跃提醒删除）；演示环境可每次重置。  
- **删除权**：按 `subject_id` **级联删除**或匿名化（Neo4j 中 DETACH DELETE 子图；对象存储删除原文件）。  
- **导出**：提供机器可读导出（JSON 子图）便于用户迁移；导出同样受鉴权约束。

---

## 5. 透明度与可解释性

- UI 区分：**观测层**（交互、话语）与 **推断层**（`TraitHypothesis`）。  
- 每条重要推断应可导航至 **Evidence → Utterance**（或量表条目）。  
- 模型与抽取器版本号写入节点属性，便于复现与争议核查。

---

## 6. 安全与访问控制

- 多租户隔离（`tenant_id`）；管理员审计日志。  
- 传输 HTTPS；存储加密（按部署环境）。  
- API 速率限制与异常监控，防止批量爬取。

---

## 7. 研究与伦理审查

若用于学术收集人类被试数据，应另行通过**机构伦理委员会（IRB）**审查；本产品文档不替代伦理批件。
