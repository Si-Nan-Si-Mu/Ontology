# 人格本体图 POG — 答辩

**产品名（工作名）**：Personality Ontology Graph（POG）  
**一句话**：把多模态「关于我」的材料，变成**可查询、可审计、可迭代**的 Neo4j 人格—社会图，再叠加 AI 做摘要与情境推演。

---

## 问题

LLM 直接「猜人格」：**难复现、难合并多源数据、难解释证据链**，且不利于产品化权限与合规。

## 解法

- **本体（TBox）**：统一「人—身份—关系—事件—话语—假设—证据」语义。  
- **图数据库（Neo4j）**：固化**时间轴 + 社会结构**，支持路径查询与版本化假设节点。  
- **AI 角色**：抽取与生成；**图与规则**：校验、冲突、置信度——**生成—约束—反馈**。

## 创新点（30 秒内）

1. **人格即图**：不是单一分数向量，而是带**证据锚定**的 `TraitHypothesis` 子图。
2. **工程可卖**：B2B API（写入/查询）、教练工作台插件、企业**社工防御**演练数据层。
3. **对齐学界**：人格知识图谱、文本+KG 预测、自我中心网络与大五等方向已有研究基础（详见 `PROJECT_BRIEF.md` 参考文献）。

## 演示路径（MVP）

虚构人物数据 → 导入聊天记录/文本 → 自动建图 → **图可视化 + 证据侧栏** → 固定 Cypher 查询（冲突链、某特质假设及证据）→（可选）GraphRAG 生成**带引用**的摘要。

## 商业与扩展

- 计费锚点：活跃档案数 / API / 存储 / 私有化。  
- 场景：心理科技辅助整理、NPC 关系记忆、合规培训；**不做医疗诊断声明**。

## 伦理边界（必讲）

仅处理**本人授权或虚构演示**数据；禁止未授权他人画像与攻击性社工；输出区分**观测事实**与**模型假设**。
